# Курсовая работа
**Создание персонального сайта с использованием MkDocs**

Ссылка на сайт:  
https://markoliks.github.io/2025_2026-introduction-in-web-tech-U4125-popov_m_a/

## Цель работы

Создать персональный сайт с использованием MkDocs и Markdown, настроить навигацию, оформить страницы, добавить изображения и подготовить сайт к публикации.

## Ход работы

### Этап 1. Подготовка и установка

На первом этапе был установлен Python, после чего выполнена установка MkDocs и темы Material.

Использованные команды:

```bash
pip install mkdocs
pip install mkdocs-material
mkdocs --version

Далее был создан новый проект MkDocs, изучена его структура и подготовлена рабочая директория.

Этап 2. Настройка конфигурации

В файле mkdocs.yml были настроены основные параметры сайта:
 • site_name
 • site_description
 • site_author
 • site_url

Также была подключена тема Material и настроены функции навигации:
 • navigation.tabs
 • navigation.sections
 • navigation.top
 • search.highlight
 • search.share

Была создана навигация со следующими разделами:
 • Главная
 • О себе
 • Проекты
 • Контакты

Этап 3. Создание контента

Для сайта были созданы страницы:
 • index.md
 • about.md
 • projects.md
 • contacts.md

На главной странице размещено приветствие и краткая информация.
На странице «О себе» добавлены сведения об образовании, навыках и интересах.
На странице «Проекты» размещены примеры выполненных проектов.
На странице «Контакты» указаны способы связи.

При оформлении использовались элементы Markdown:
 • заголовки
 • списки
 • ссылки
 • изображения
 • цитаты
 • таблицы

Этап 4. Дополнительные возможности

Была создана папка docs/images/ для изображений.
На главную страницу добавлено изображение.
Также была протестирована работа поиска и корректность переходов между страницами.

Этап 5. Стилизация и кастомизация

Была подключена тема Material, выбрана цветовая схема и улучшена визуальная навигация сайта.
Сайт был оформлен как персональная страница-портфолио.

Этап 6. Тестирование и публикация

Для локального тестирования использовалась команда:

mkdocs serve

Для сборки сайта использовалась команда:

mkdocs build

Для публикации сайта на GitHub Pages была настроена публикация через ветку gh-pages.

Результат

В результате работы был создан персональный сайт на MkDocs с несколькими страницами, навигацией, оформлением и публикацией через GitHub Pages.
